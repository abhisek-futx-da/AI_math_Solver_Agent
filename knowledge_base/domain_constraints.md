# Domain and Unit Constraints

## Real-world constraints
- Probabilities: 0 ≤ p ≤ 1; counts non-negative integers.
- Length, area, volume: non-negative.
- Rates (e.g. speed): often non-negative unless direction specified.

## Algebraic domain
- √(g(x)): g(x) ≥ 0
- 1/g(x): g(x) ≠ 0
- log_a(g(x)): g(x) > 0, a > 0, a ≠ 1
- Rational: denominator ≠ 0

## Calculus domain
- f(x) must be defined on the interval; check endpoints.
- For "maximum area" etc., variable (e.g. length) often in (0, L) or [0, L].

## Units
- Keep units consistent (e.g. all in meters or all in cm).
- Final answer: state units when applicable (e.g. "area = 12 cm²").
